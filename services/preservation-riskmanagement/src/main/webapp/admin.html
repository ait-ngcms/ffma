Component Installation

To use the service utilize following links.
These links accomplish database collection initialization tasks:

  • Retrieval of overall format information from Linked Open Data (LOD) repositories like Freebase, Pronom, DBPedia and AIT
  • Retrieval of rich data for aggregated formats and creation of LODFormat, LODSoftware and LODVendors collections
  • Mapping creation between file formats, associated software and vendors. This mapping is stored in collectins DipFormatId, DipSoftwareId and DipVendorId

Component Testing

To use the service utilize following links.
These links accomplish database collection testing tasks:

  • Checking of database collections existence. The result of the method should contain following collections: PronomFileFormat, DBPediaFileFormat, FreebaseFileFormat, LODFormat, LODSoftware, LODVendor, DipFormatId, DipSoftwareId, DipVendorId In case of missing collections fill missing data applying storeAllExtensions() method. Internet connection is necessary for this step to enable MQL, SPARQL and HTTP queries.
